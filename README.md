## Описание проекта

В проекте разработан полный ML-пайплайн для анализа музыкального датасета Spotify и классификации треков по сценариям использования.

Основная задача — определить, для какого типа плейлиста подходит музыкальный трек:

- `Good for Party`
- `Good for Work/Study`
- `Good for Relaxation/Meditation`
- `Good for Driving`

Проект включает загрузку и подготовку данных, EDA-анализ, разметку, препроцессинг, обучение нескольких моделей машинного обучения, выбор лучшей модели и сохранение результатов пайплайна.

**Тип задачи:** multi-output binary classification  
**Лучшая модель:** LightGBM  
**Основная метрика выбора модели:** f1_macro  

---

## Бизнес-задача

Музыкальные сервисы используют рекомендации и автоматическую классификацию треков, чтобы формировать плейлисты под настроение и контекст пользователя.

Например, один и тот же трек может подходить для вечеринки, поездки на машине или работы. Ручная разметка таких сценариев занимает много времени, поэтому возникает задача автоматизировать процесс.

**Цель проекта** — построить ML-пайплайн, который по аудиохарактеристикам и метаданным трека определяет, для каких сценариев использования он подходит.

**Потенциальный заказчик:** музыкальный стриминговый сервис, рекомендательная система, сервис автоматической генерации плейлистов.

## Этапы проекта

### 1. Загрузка и подготовка данных

На первом этапе были загружены исходные данные Spotify и выполнена базовая очистка.

**Результаты обработки:**

| Метрика | Значение |
|---|---|
| Исходные данные | 100 000 строк × 25 колонок |
| Финальные данные после очистки | 85 593 строк × 38 колонок |
| Удалено дубликатов | 14 403 |
| Заполнено пропусков | 10 значений |
| Обработано выбросов | 49 257 значений |

### 2. Преобразование признаков

В пайплайне были выполнены следующие преобразования:

| Исходный признак | Новый признак |
|---|---|
| `Length` | `length_seconds` |
| `Release Date` | `release_year`, `release_month` |
| `Loudness (db)` | `loudness_db` |
| `Explicit` | `explicit_int` |
| `Key` | `key_note`, `key_mode` |
| `Time signature` | `time_sig` |
| `Genre` | бинарные жанровые признаки |

Также были созданы дополнительные признаки:

| Признак | Описание |
|---|---|
| `energy_acoustic_ratio` | показатель электронности трека |
| `dance_mood_index` | танцевальность × позитивность |
| `track_age` | возраст трека |
| `is_speech_heavy` | флаг речевых треков |

## EDA-анализ

На этапе разведочного анализа были изучены:

- структура датасета;
- типы признаков;
- пропущенные значения;
- распределение аудиохарактеристик;
- распределение эмоций;
- качество и баланс целевых переменных;
- потенциальные задачи машинного обучения.

### Основные наблюдения

1. В исходных данных были признаки, которые требовали преобразования типов: `Length`, `Loudness (db)`, `Time signature`.
2. Текстовое поле `text` содержит большой объем данных, поэтому для базового ML-пайплайна оно не использовалось как основной признак.
3. Теги `Good for ...` подходят для задачи multi-label classification.
4. Датасет содержит как числовые признаки, так и категориальные признаки, которые требуют кодирования.

## Обучение моделей

Для решения задачи были протестированы несколько моделей:

| Модель | Обоснование выбора |
|---|---|
| Random Forest | Базовая устойчивая модель для табличных данных |
| LightGBM | Быстрый и эффективный градиентный бустинг |
| LightGBM Chain | Модель, учитывающая зависимости между несколькими целевыми метками |

Данные были разделены на:

- train — для обучения;
- validation — для выбора лучшей модели;
- test — для финальной оценки качества.
